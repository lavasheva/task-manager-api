# Task Manager API

REST API для управления задачами, разработанное на Java с использованием
Spring Boot, Spring Data JPA и PostgreSQL.

Проект создан в качестве pet‑проекта для практики разработки
backend‑приложений и REST API.

------------------------------------------------------------------------

# Функциональность

API поддерживает основные CRUD операции:

-   создание задачи
-   получение списка задач
-   получение задачи по id
-   обновление задачи
-   удаление задачи

Также реализовано:

-   валидация входящих данных
-   обработка ошибок через глобальный обработчик исключений
-   работа с базой данных PostgreSQL через JPA/Hibernate

------------------------------------------------------------------------

# Используемые технологии

-   Java
-   Spring Boot
-   Spring Web
-   Spring Data JPA
-   Hibernate
-   PostgreSQL
-   Maven
-   Postman (для тестирования API)

------------------------------------------------------------------------

# Структура проекта

src/main/java/com/example/taskmanager

controller --- REST контроллеры\
service --- бизнес‑логика приложения\
repository --- слой доступа к данным\
model --- сущности и модели данных\
exception --- обработка ошибок и исключений

------------------------------------------------------------------------

# Конфигурация базы данных

Пример настройки в `application.properties`:

spring.datasource.url=jdbc:postgresql://localhost:5432/taskdb\
spring.datasource.username=postgres\
spring.datasource.password=your_password

spring.jpa.hibernate.ddl-auto=update\
spring.jpa.show-sql=true\
spring.jpa.properties.hibernate.format_sql=true

------------------------------------------------------------------------

# API эндпоинты

### Получить все задачи

GET /tasks

### Получить задачу по id

GET /tasks/{id}

### Создать задачу

POST /tasks

Пример тела запроса:

{ "title": "Изучить Spring Boot", "description": "Создать первый backend
pet‑проект", "status": "TODO" }

### Обновить задачу

PUT /tasks/{id}

### Удалить задачу

DELETE /tasks/{id}

------------------------------------------------------------------------

# Пример ответа API

{ "id": 1, "title": "Изучить Spring Boot", "description": "Создать
первый backend pet‑проект", "status": "TODO" }

------------------------------------------------------------------------

# Валидация данных

В проекте используется валидация входящих данных.

Примеры:

-   `title` не может быть пустым
-   `status` обязателен

Если данные некорректные, сервер возвращает ошибку `400 Bad Request`.

------------------------------------------------------------------------

# Обработка ошибок

В проекте реализован глобальный обработчик ошибок
(`GlobalExceptionHandler`).

Пример ответа при отсутствии задачи:

{ "timestamp": "2026-03-09T11:35:15", "status": 404, "error": "Not
Found", "message": "Task with id 400 not found" }

------------------------------------------------------------------------

# Как запустить проект

1.  Клонировать репозиторий
2.  Открыть проект в IntelliJ IDEA
3.  Настроить подключение к PostgreSQL в `application.properties`
4.  Запустить `TaskmanagerApplication`
5.  Использовать Postman для тестирования API
